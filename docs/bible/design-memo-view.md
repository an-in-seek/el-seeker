# 마이페이지 — 내가 작성한 성경 구절 메모 조회

## 1. 기능 목적

성경 읽기 중 작성한 구절 메모를 마이페이지에서 **한눈에 모아 볼 수 있는 기능**을 제공한다.

현재 메모는 성경 구절 화면(`/web/bible/verse`)에서 개별 장 단위로만 확인 가능하다.
사용자가 여러 책·장에 걸쳐 남긴 메모를 통합 조회하고, 원래 구절 화면으로 바로 이동할 수 있는 흐름이 없다.

핵심 가치: **흩어진 메모를 한곳에서 확인하고, 해당 구절로 즉시 돌아갈 수 있는 경험**

---

## 2. 사용자 흐름

```
마이페이지 진입
  └─ "내 메모" 섹션 (또는 탭) 확인
       └─ 메모 목록 조회 (최신순 기본)
            └─ 메모 카드 선택
                 └─ 해당 성경 구절 화면으로 이동
                    (/web/bible/verse?translationId=..&bookOrder=..&chapterNumber=..)
```

- 마이페이지 내 별도 섹션으로 배치한다. 기존 "내 정보 수정 / 연동 계정 / 회원탈퇴" 아래에 추가하거나, 마이페이지를 탭 구조로 확장하는 방식을 검토한다.
- 인증된 사용자만 접근 가능하다. (마이페이지 자체가 인증 필요)

---

## 3. 화면 방향

### 3-1. 메모 목록

| 항목 | 내용 |
|------|------|
| **위치** | 마이페이지 하단 섹션 또는 별도 탭 |
| **카드 구성** | 책 이름 · 장:절 정보, 메모 내용 미리보기(2~3줄), 작성/수정일 |
| **정렬** | 최신 수정순 (기본) |
| **페이징** | Slice 기반 "더보기" 또는 무한 스크롤 (기존 커뮤니티 패턴 활용) |
| **빈 상태** | "아직 작성한 메모가 없습니다. 성경 읽기에서 메모를 남겨보세요." + 성경 읽기 바로가기 |

### 3-2. 모바일 대응

- 카드형 리스트로 구성하여 모바일에서도 자연스럽게 표시한다.
- 책 이름·장:절 정보는 한 줄로 간결하게, 메모 내용은 말줄임(…) 처리한다.
- 기존 마이페이지 반응형 breakpoint(`768px`, `576px`)를 동일하게 적용한다.

### 3-3. 구절 이동 링크

- 각 메모 카드를 클릭하면 해당 구절의 성경 읽기 화면으로 이동한다.
- URL 형식: `/web/bible/verse?translationId={translationId}&bookOrder={bookOrder}&chapterNumber={chapterNumber}`
- 이동 후 해당 절이 하이라이트되거나 스크롤 위치가 맞춰지면 이상적이나, 초기에는 장 단위 이동만으로 충분하다.

## 4. 데이터 조회 방향

### 4-1. 현재 상태

- `BibleVerseMemo` 엔티티에 `member`, `translationId`, `bookOrder`, `chapterNumber`, `verseNumber`, `content` 저장 중
- 기존 Repository 쿼리는 **특정 장 단위 조회** (`findAllByMemberUidAndTranslationIdAndBookOrderAndChapterNumber`)만 존재
- **사용자 전체 메모 조회 쿼리가 없음** → 신규 추가 필요

### 4-2. 필요한 조회

| 쿼리 | 용도 |
|------|------|
| `findAllByMemberUid(memberUid, Pageable): Slice<BibleVerseMemo>` | 사용자의 전체 메모를 페이징 조회 |

- 정렬: `updatedAt DESC` (최신 수정순)
- Slice 기반으로 `hasNext` 여부만 확인 (전체 count 쿼리 불필요)
- 책 이름 표시를 위해 `BibleBook` 정보 조인 또는 별도 조회 필요
  - 방법 A: JPQL로 `BibleBook`을 JOIN하여 책 이름을 함께 반환
  - 방법 B: 메모 목록 조회 후, `bookOrder` + `translationId` 기준으로 책 이름을 별도 조회 (캐시 활용 가능)
  - 초기에는 **방법 B**가 단순하다. `BibleVerseMemo`는 `BibleBook`과 직접 연관관계가 없으므로 JOIN 시 추가 매핑이 필요하기 때문.

### 4-3. API 엔드포인트

```
GET /api/v1/bibles/my-memos?page=0&size=20
```

- 기존 헥사고날 모듈 경계를 유지하여 bible 모듈(`/api/v1/bibles/...`) 하위에 배치한다.
  - 대안: `/api/v1/members/me/bible-memos` (사용자 중심 경로). 단, 현재 프로젝트에 모듈 간 경로를 넘는 사례가 없으므로 bible 모듈 내 배치를 우선 고려한다.
- 응답: `{ content: [...], hasNext: boolean, size: int, number: int }`
- 각 항목: `{ memoId, translationId, bookOrder, bookName, chapterNumber, verseNumber, content, updatedAt }`

## 5. 향후 확장 방향

### 단기 (현재 기능 안정화 이후)

- **메모 삭제**: 목록에서 스와이프 또는 삭제 버튼으로 개별 삭제
- **메모 수정**: 목록에서 인라인 수정 또는 모달 편집
- **절 단위 스크롤**: 구절 이동 시 해당 `verseNumber` 위치로 자동 스크롤

### 중기

- **필터/검색**: 책 단위 필터, 메모 내용 키워드 검색
- **번역본 표시**: 메모가 어느 번역본 기준인지 표시 (현재 KRV 위주이므로 초기에는 생략 가능)
- **메모 내보내기**: 텍스트 또는 이미지로 메모 공유

### 장기

- **하이라이트 통합 조회**: 메모와 하이라이트를 함께 "내 성경 활동" 으로 통합
- **태그/카테고리**: 메모에 태그를 붙여 주제별 분류

## 6. 구현 범위 제안

**1차 구현 (MVP)**
- Repository: 사용자 전체 메모 Slice 조회 쿼리 추가
- Service: 메모 목록 조회 로직
- API: `GET /api/v1/bibles/my-memos` 엔드포인트
- 화면: 마이페이지 하단에 메모 목록 섹션 추가 (카드형, 더보기 버튼)
- 빈 상태: "아직 작성한 메모가 없습니다" 안내 + 성경 읽기 바로가기
- 구절 이동: 카드 클릭 시 성경 읽기 화면으로 이동

**2차 구현**
- 메모 삭제 기능
- 절 단위 스크롤 이동
